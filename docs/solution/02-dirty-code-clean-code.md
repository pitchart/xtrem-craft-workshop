# Clean code VS Dirty code

## Qu'est-ce que du code sale ?

- C'est un code qui ne repond pas aux principes SOLID. C'est un ensemble de regles de codes qui permettent d'avoir un code evolutif et maintenable et reconnu par la communauté des développeurs. 

- C'est un code qui utilise de nombreux commmentaires pour décrire un concept. Cela signifie que le concept n'est même pas compris par le développeur parfois. 

- C'est un code qui utilise des noms de variables non explicites et qui provoquent des incomprehensions. 

- Les conditions dans le code qui ne permettent pas de comprendre ce qui est testé. 

- L'abscence de documentation pour expliquer le fonctionnement de concepts ou de programmes ou technologies. 

- Les noms de commits qui ne suivent pas une charte prédéfinie. 

- Un environnement de développement sur git mal organisé sans avoir de branch de production, préproduction, etc. 

## Qu'est-ce que du code propre ?

- Du code documenté, par rapport aux entrées, aux responsabilités et aux sorties de chaque méthode et classes
- Des tests explicites et conformes au besoin exprimé initialement
- De la documentation par rapport aux processus d'installation, de configuration de l'application, de gestion des accès aux environnements...
- De la veille sur les librairies et outils utilisés et de la surveillance de performance, sécurité et evolutivité.
