# Clean code VS Dirty code

## Qu'est-ce que du code sale ?
- Un code incompréhensible (copié collé, IA )

- Pas de commentaires

- Pas optimisé (duplications, code mort), pas structuré, classes super longues

- Conventions pas respectées

- requirements mal écrits

- Pas relu par soi-même et les autres

- Code sans mises à jour

## Qu'est-ce que du code propre ?

- Historique de versionning est clair (commit + push)

- Commentaires explicites et concis

- Convention citée dans le readme (tout le monde est au courant) et respectée
